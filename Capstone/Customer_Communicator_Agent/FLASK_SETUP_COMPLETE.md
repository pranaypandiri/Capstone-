## 📊 What Each Endpoint Does

### GET /health
- **Purpose:** Check API is running
- **Response:** Status, timestamp, agent ready status
- **Use:** First verification test

### GET /api/v1/status
- **Purpose:** Get agent configuration
- **Response:** Agent ID, status, available endpoints
- **Use:** Verify agent initialization

### POST /api/v1/generate-message
- **Purpose:** Generate ONE personalized message
- **Process:**
  1. Receives customer, resolution, credit data
  2. Calls Message Generator Agent → LLM creates personalized message
  3. Calls Compliance Validator Agent → LLM validates for GDPR/brand
  4. Returns complete message + validation
- **Response:** Message body, compliance status, dispatch channel
- **Terminal:** Shows agent invocation logs

### POST /api/v1/batch-generate
- **Purpose:** Generate MULTIPLE messages at once
- **Process:**
  1. Receives array of customer requests
  2. Processes each message through agents
  3. Returns all results with success/failure status
- **Response:** Total count, succeeded count, results array
- **Terminal:** Shows progress for each message

### POST /api/v1/validate
- **Purpose:** Validate message compliance
- **Process:**
  1. Receives message and context
  2. Calls Compliance Validator Agent
  3. Returns GDPR/brand/regulatory check results
- **Response:** Compliance status, issues, suggestions
- **Terminal:** Shows validation process

### Batch Processing
```python
import requests

batch_request = {
    'messages': [
        {...message1...},
        {...message2...},
        {...message3...}
    ]
}

    'http://127.0.0.1:5000/api/v1/batch-generate',
    json=batch_request
)
print(f"Processed: {response.json()['succeeded']}/{response.json()['total']}")
```
